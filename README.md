# News Briefing

개인용 뉴스·증권 브리핑 자동화 시스템.

## 이 프로젝트가 하는 일

매일 아침 출근 시간에 국내외 뉴스와 주요 공시를 요약해서 카카오톡으로 받습니다. 알림에는 "이 공시가 왜 의미있는지" 자동 설명이 붙고, 시간이 지나면 트렌드에서 수혜 섹터와 상장사를 분석하는 기능까지 확장됩니다.

## 문서 가이드

구현에 들어가기 전에 아래 순서로 읽으면 전체 그림이 잡힙니다.

1. [`docs/PRD.md`](docs/PRD.md) — 이 시스템이 해결하는 문제, 기능 요구사항, 의도적으로 하지 않는 것
2. [`docs/ARCHITECTURE.md`](docs/ARCHITECTURE.md) — 시스템 구성, 데이터 흐름, 기술 스택 선택 이유
3. [`docs/ROADMAP.md`](docs/ROADMAP.md) — 4주 개발 계획, 주차별 완료 기준
4. [`docs/SIGNALS.md`](docs/SIGNALS.md) — 공시 점수 체계, 시그널 해석 템플릿, 밸류체인 분석 접근법 (도메인 지식)
5. [`docs/DESIGN.md`](docs/DESIGN.md) — UI/UX 디자인 시스템, 색상·타이포·컴포넌트·다국어·다크모드·접근성
6. [`docs/DECISIONS.md`](docs/DECISIONS.md) — 주요 설계·제품 결정 로그 (왜 자동매매는 없는지, 왜 Max 플랜 CLI인지, 왜 Pretendard인지 등)
7. [`CLAUDE.md`](CLAUDE.md) — Claude Code로 개발할 때의 프로젝트 컨텍스트와 원칙

## 핵심 원칙 (꼭 숙지)

이 시스템은 **매매 추천 시스템이 아닙니다.** 특정 종목을 사라/팔라고 말하지 않습니다. 대신:

- 놓치면 안 되는 **이벤트와 시그널을 수면 위로** 띄우고
- **개념을 모르는 사람도 이해할 수 있게 용어를 자동 해설**하고
- **트렌드가 포착되면 밸류체인으로 분해**해 후보 상장사들을 보여주되, 투자 결정은 본인이 합니다

이 경계는 `CLAUDE.md`와 `docs/SIGNALS.md`에 더 자세히 나와있습니다. 구현 중 이 선을 넘지 않도록 주의하세요.

## 개발 시작

코드는 Claude Code에서 개발합니다. 프로젝트 루트에서:

```bash
claude
```

실행하면 Claude Code가 `CLAUDE.md`와 `docs/`를 읽어 프로젝트 컨텍스트로 삼습니다. 첫 작업은 `docs/ROADMAP.md`의 Week 1부터 시작하세요.
