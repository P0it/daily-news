# 제품 요구사항 (PRD)

## 1. 배경과 문제

### 1.1 왜 만드는가

개인 투자자 입장에서 아침 브리핑이 제대로 정리되면 하루의 출발이 달라진다. 특히:

- **정보원이 흩어져 있다.** DART 공시, 경제 뉴스, 해외 증시, 일반 시사를 각각 확인하려면 여러 앱·사이트를 떠돌게 된다. 매일 아침 정제된 한 페이지가 필요하다
- **정보는 많은데 해석이 어렵다.** "내부자 매수", "전환사채 발행", "최대주주 변경" 같은 용어가 무엇을 의미하고 어떤 신호인지 일반인은 모른다
- **테마가 떠도 수혜주를 모른다.** "로봇이 뜬다"는 건 알겠는데 테슬라, 현대차 외에 누가 수혜인지, 공통분모 섹터가 무엇인지 찾으려면 리서치가 필요하다
- **매일 아침 출근길에 시장 브리핑을 읽고 싶다.** 밤사이 공시·뉴스를 요약받아 출근길에 훑어보는 경험을 만든다

### 1.2 사용자

본인(개인 투자자) 1명. 구독 서비스가 아니라 개인 자동화 시스템이다.

### 1.3 성공 기준

- **아침 6:00에 카카오톡이 온다.** 출근길에 읽을 수 있는 요약 링크
- **밤사이 + 이른 아침 주요 공시가 브리핑에 포함된다.** 전일 장 마감(15:30) 이후 및 당일 아침 시간의 중요 공시를 놓치지 않는다
- **알림을 읽고 "이게 뭐지"가 없다.** 용어 자동 해설이 있어서 검색 없이 이해된다
- **주 1회, "이번 주 뜨는 테마와 수혜 섹터"를 리포트로 받는다** (Week 4 이후)
- **웹 페이지를 열었을 때 5초 안에 오늘의 핵심이 보인다.** 점수별 시각적 계층으로 스캔 가능 (상세 기준은 `DESIGN.md` 참조)

### 2.5 의도적 Non-goals (하지 않는 것)

명시적으로 **구현하지 않는다.**

- ❌ 특정 종목 매수·매도 추천 (예: "삼성전자 지금 사세요")
- ❌ 가격 예측 모델 (내일 몇 % 오른다 등)
- ❌ 손절 라인 알림 (구체적 가격 기반 매도 신호)
- ❌ **자동 매매 연결** — 시그널 감지 시 증권사 API로 주문을 자동 집행하는 기능. 이 결정의 상세 근거는 [`DECISIONS.md`](DECISIONS.md) 1번 항목 참조. 재고 조건도 그 문서에 명시.
- ❌ **장중 실시간 폴링·긴급 알림** — 60초 간격 DART 폴링으로 공시 즉시 알림. 이 프로덕트는 **아침 배치 브리핑**에 집중. 실시간 트레이딩 도구 아님
- ❌ 타인에게 제공·공유 (투자자문업 이슈)
- ❌ 장중 단타용 ms 단위 속보 (블룸버그 터미널 영역)

**허용되는 "매매 실행 편의 기능"** (자동매매와 구분):
- ✅ 증권사 앱 딥링크 — 주문 화면까지 자동 이동, 체결은 사용자 손 (F19)
- ✅ 차트 임베드 — 판단 재료 제공 (F18)

이 경계는 `CLAUDE.md`와 `docs/SIGNALS.md`에서 상세히 다룬다. 예측이 아닌 **교육·분석·시그널 surfacing**이 이 시스템의 역할이다.

## 3. 비기능 요구사항

### 3.1 신뢰성

- **조용한 실패 방지**: 새벽 실행 시 오류가 나도 다음 날 정상 동작 보장 (한 소스가 죽어도 전체가 멈추지 않음)
- **알림 중복 방지**: 같은 공시가 여러 경로로 들어와도 한 번만 알림
- **토큰 자동 관리**: 카카오 access_token 만료 시 refresh_token으로 자동 갱신

### 3.2 비용

- **LLM 비용 $0 목표**: 기존 Claude Max 플랜 ($100/월) 할당량 내에서 해결
- **인프라 $0 목표**: 로컬 맥에서 실행 (클라우드 서버 불필요)
- **외부 API 무료 티어 활용**: DART(무료), 카카오(무료)
- Week 3 이후 확장 시에만 유료 API 고려 (NewsAPI, Polygon 등)

### 3.3 성능 목표

- 아침 브리핑 end-to-end < 3분 (수집 → 요약 → 발송)
- 06:00 launchd 트리거 이후 06:03 이내 카톡 수신 완료
- LLM 요약 캐시 히트율 > 50% (같은 내용 재요청 시 LLM 호출 없음)
- 주간 리포트 생성 (Week 4+) < 5분

### 3.4 프라이버시

- 모든 상태 데이터는 로컬 SQLite (외부 DB 없음)
- API 키는 `.env` (gitignore), 카카오 토큰은 `.kakao_tokens.json` (gitignore)
- 분석 대상은 모두 공개 데이터 (뉴스·공시)라 LLM API 전송해도 무방
- Claude API / Claude Code 사용 시 정책에 따라 데이터 처리 (Anthropic 정책 참조)

## 4. 제약사항과 가정

### 4.1 기술 제약

- **개발 환경**: macOS + VSCode + Python
- **LLM 주 엔진**: Claude Code CLI (Max 플랜 호출)
- **LLM 보조 엔진**: Ollama 로컬 (이미 설치됨, Max 한도 초과 대비)
- **카카오 개인 용도**: 사업자 등록 없음 → "나에게 보내기" API 사용 (알림톡 불가)

### 4.2 가정

- 사용자가 매일 아침 카카오톡을 확인한다
- macOS가 새벽에 sleep 상태여도 scheduled job이 실행된다 (launchd 활용)
- 한국 시장 기준 (장 시간 09:00 ~ 15:30 KST)

## 5. 확장 계획 (Out of Scope for v1)

이 4주 로드맵 완료 후 고려할 항목:

- 텔레그램 채널 병행 (카톡 4KB 제한 회피)
- 음성 브리핑 (출근 운전 중)
- 다국어 해외 뉴스 (일본, 중국)
- 포트폴리오 연동 (보유 종목 관련 뉴스 우선)
- **시그널 백테스트 모듈** — 과거 시그널 발생 후 주가 움직임 통계. 시그널 체계 유효성 검증. 자동매매 재검토 전제 조건
- **페이퍼 트레이딩 모드** — 가상 자금으로 시그널 전략 성과 측정 (최소 3~6개월)
- 한국투자증권 KIS API 기반 실시간 시세 (F20)
